# 미션 (Mission)

## 왜 이것을 학습하고 있는가?

저는 중국 중고차 거래 회사에 소속되어 있으며, 현재 상장(IPO)을 준비하고 있습니다. 미국 시장에서 동일 트랙에 이미 상장된 경쟁사들의 동향을 모니터링해야 하며, 이를 전략적 투자의 참고 자료로 활용하고자 합니다. 즉, 경쟁사의 재무 상태, 전략적 행동, 위험 신호를 파악하여 우리 회사의 상장 일정과 사업 전략을 조율해야 합니다.

저는 **이 모니터링 시스템의 배경이 되는 도메인 지식**(코딩 구현이 아님)을 숙지해야 합니다. 그 정도는 다음과 같습니다. 전략 투자부의 동료들에게 이 시스템의 각 구성 요소가 무엇을 하는지, 왜 이렇게 설계되었는지, 산출된 결과가 무엇을 의미하는지를 설명할 수 있을 정도입니다.

## 무엇을 할 수 있게 될 것인가?

- SEC filing의 유형, 구조, 핵심 정보를 읽고 이해하며, 어디에서 무엇을 찾아야 하는지 알 수 있다
- 중고차 트랙 경쟁사 모니터링의 핵심 논리를 이해한다. 누구를 모니터링하고, 어떤 지표를 보며, 왜 그러한가
- 시스템이 산출한 대시보드나 회의록을 받은 후, "이 정보가 중요한지, 상부 보고가 필요한지" 판단할 수 있다
- 전략 투자부의 내부 회의에서 이 모니터링 시스템의 작동 방식과 산출 가치에 대해 설명할 수 있다

## 무엇을 학습하지 않는가?

- Python 프로그래밍, Streamlit 개발, API 호출 — 이미 할 수 있다
- LLM 프롬프트 엔지니어링 — 코딩 에이전트에게 맡긴다
- SEC 법률 컴플라이언스 세부 사항 — 이것은 법률 강좌가 아니다

## 성공 기준 (Success criteria)

- "SEC를 모르는" 동료에게 10-K, 10-Q, 8-K가 각각 무엇이며 왜 모두 중요한지 독립적으로 설명할 수 있다
- 우리가 어떤 5개 경쟁사를 선택했는지, 왜 이 5곳인지, 각사의 핵심 모니터링 포인트는 무엇인지 설명할 수 있다
- 8-K 경고(예: "Item 5.02 임원 퇴임")를 접했을 때, 다른 사람에게 묻지 않고 그것이 얼마나 중요한지 안다
- Earnings Call 회의록과 10-Q의 핵심 데이터를 읽고, 5분 안에 자신의 판단을 형성할 수 있다
