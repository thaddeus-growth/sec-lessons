13F 与机构持仓
交叉持股分析的数据底座

前面 5 课讲的都是公司自己披露的信息——10-K、10-Q、8-K、Earnings Call 都是公司角度。这一课开始讲第三方对公司的看法——谁在买、谁在卖、谁在重仓、谁在撤资。这是 IHS Markit 报告（P4 交叉持股分析）的核心数据源。

13F 是美国 SEC 强制要求大型机构投资者披露持仓的报表，全称 Schedule 13F。它是公开的、免费的、每季度更新的，因此成为交叉持股分析的事实标准。

IHS Markit 那份参考报告的 P2-P7 几乎全部建立在 13F 数据之上。我们的系统用的也是 13F——只是数据源从 IHS 切到了 SEC EDGAR。

13F 是什么

把 13F 当作"机构投资者的季度成绩单"——但有几条硬规则要知道：

规则具体要求战略含义
法律基础Securities Exchange Act of 1934, Section 13(f)法定披露，不提交会违法
申报门槛管理资产 > $1 亿的机构投资者覆盖了几乎所有大型共同基金、对冲基金、养老金、ETF 发行人
申报频率每季度结束后 45 天内Q1 截止 5/15、Q2 截止 8/14、Q3 截止 11/14、Q4 截止 2/14
申报机构数约 5,000-6,000 家基本覆盖美国市场的"聪明钱"
13F 反映的是季度末的持仓快照——不是季度内的交易。一只股票可能在 3 月被买入、4 月被卖出，但 13F 上完全看不到。IHS Markit 报告的 Turnover 列是基于相邻两期 13F 推算的"调仓幅度"，不是真实的交易流水。

13F 披露什么、不披露什么

了解 13F 的"边界"比知道它披露什么更重要——因为这些边界直接决定了它能回答什么问题、不能回答什么问题。

披露的内容（13F 里有什么）

  • 每只持仓股票的：名称、CUSIP、股数、市值
  • 市值字段叫 value_x1000，单位是千美元（不是美元，也不是百万美元）
  • 覆盖所有美国注册证券的多头仓位

不披露的内容（13F 的盲区）

看不到为什么看不到对分析的影响
做空仓位13F 只报多头对冲基金的空头押注完全缺失——而这可能是最强信号
期权/衍生品13F 不含衍生品很多对冲基金通过期权表达观点，13F 上看不到
美国以外的持仓13F 只覆盖美股全球机构在非美股上的仓位完全不可见
非上市证券私募债、未上市公司不在 13F 范围PE 基金、private credit 基金的活动看不到
季度内调仓只显示季末快照日内、周内、月内的交易全部被平均成"季末持有"
申报机构 < $1 亿的部分不达门槛不报小机构和散户行为不在 13F 覆盖范围
记住一句话：13F 看到的是"季度末这张照片"，不是"这一季度的电影"。如果想看电影，需要更细的数据（如 Bloomberg 的交易流水、WhaleWisdom 高级订阅）。

单位转换陷阱（出 bug 的高发区）

13F 数据处理里最容易出错的地方是单位转换。下面这张表是真实出 bug 的模式：

SEC 原始 XML: <value>12345</value>  →  含义: $12,345,000  →  $12.345M
edgartools 返回: 字段名 value_x1000 → 数值 12345 → 含义: 12345 × $1,000

常见错误:
  错误 A: 把 value_x1000 当成"千美元"展示 → 数字看起来是 12,345 → 实际是 $12.345M
          错位 1000 倍
  错误 B: 在 12345 之上又除以 1000 展示成 12.345 → 错位 1,000,000 倍
  错误 C: 汇总时不区分市值量级
          → Vanguard 在 CVNA 持仓 $1.2B（value_x1000 = 1,200,000）
          → BlackRock 在 UXIN 持仓 $3M（value_x1000 = 3,000）
          → 如果直接 SUM，UXIN 的变动被 CVNA 淹没（差 400 倍）

验证方法:
  1. 取 Vanguard 在 CVNA 的 13F 原始 XML
  2. 手工读取 <value> 字段 = X
  3. 与 edgartools 解析后的 value_x1000 对比
  4. 用 CVNA 当时的股价 × 股数 交叉验证（应大致匹配，允许 ±5% 误差）

⚠️ 我们的系统如何处理

整个数据流里 value_x1000 字段保持 SEC 原始单位（千美元），只在 dashboard 展示时统一除以 1000 显示为百万美元（$M）。 所有内部计算（汇总、QoQ 变动、Top Buyers/Sellers 排名）都基于统一的 "$M" 单位，避免在数据流的不同环节混用单位。

13F 跨期对比的正确姿势

交叉持股分析的核心动作是"对比相邻两期 13F"。这一步的逻辑必须写对，否则 Top Buyers/Sellers 排名会完全失真。

边界条件（必须处理的 4 种情况）

  1. 新建仓：curr_value > 0 且 prev_value == 0 — 某机构首次出现
  2. 清仓：curr_value == 0 且 prev_value > 0 — 某机构完全退出
  3. 加仓：curr_value > prev_value — 增持
  4. 减仓：curr_value < prev_value — 减持
  5. 不变：curr_value == prev_value — 不动（罕见，股数不变+股价不变才出现）
curr = holdings WHERE report_period = '2026-03-31'
prev = holdings WHERE report_period = '2025-12-31'
merged = curr FULL OUTER JOIN prev ON (institution_cik, ticker)

for each row:
    if curr_value > 0 and prev_value == 0:  → Initiation
    if curr_value == 0 and prev_value > 0:  → Liquidation
    if curr_value > prev_value:              → Increase
    if curr_value < prev_value:              → Decrease
    if curr_value == prev_value:             → No change

注意:
  - prev_value = 0 时不要计算 delta_pct（除零）
  - FULL OUTER JOIN 后 institution_name 可能在某一侧为空，需要补全
  - 跨期对比必须是相邻两期（不是跳期），否则变动会被人为放大

对每只股票，变动 = 主动调仓 + 股价变动

如果一个机构持有 CVNA 10,000 股，本期 13F 还是 10,000 股，但股价从 $100 涨到 $150：

  • 股数没变 = 该机构没有主动买卖
  • 但市值从 $1,000,000 涨到 $1,500,000 = 纯粹是股价上涨带动的

这种"被动市值上涨"会被错误归类为"加仓"，误导 Top Buyers 排名。正确的做法是看股数变化（shares_change），而不是市值变化（value_change）。

核心原则：13F 跨期对比时，看股数（shares）而不是市值（value）来判断主动调仓。市值变化会被股价波动污染，只有股数变化才是真实的"买入/卖出"。

13F 在交叉持股分析中的角色

我们这一系列监控系统的最终产出（IHS 风格报告）的输入输出对应关系：

输入输出13F 能回答吗？
25 家种子机构 × 5 家竞品 × 2 期 13F交叉持股矩阵 (机构 × 竞品)✅ 完整
QoQ 持仓变动 (delta value / delta shares)✅ 完整（基于 shares 更准）
跨竞品 Buyers/Sellers 排名✅ 完整
Initiations & Liquidations✅ 完整
Churn Proxy (turnover 近似)⚠️ 近似（不是真实 turnover）
投资风格分类 (Value/Growth/GARP)❌ 不能（需要全持仓 + 基本面数据库）

13F 能回答"谁在买卖"，但不能回答"为什么买卖"——后者需要看管理层的 commentary（13F 不会披露交易理由）。这正是为什么需要把 13F 变动和 8-K、Earnings Call 交叉验证。

在 Carvana 上的具体例子

下表是 2026Q1（report_period 2026-03-31）我们从 EDGAR 拉到的 Carvana (CVNA) Top 5 机构持有人（按 value 排序）：

机构股数 (千股)value_x1000 ($K)折算 $M
Vanguard Group12,8001,920,000$1,920M
BlackRock10,2001,530,000$1,530M

Baupost Group3,200480,000$480M

验证单位转换：value_x1000 = 1,920,000 → 实际金额 = 1,920,000 × $1,000 = $1,920,000,000 = $1,920M = $1.92B。✓

Vanguard 的 $1.92B 持仓，是 UXIN（市值小一个数量级）所有机构持仓总和的几十倍。这就是为什么 汇总时必须按机构分别排名，不能简单 SUM——否则 UXIN 的信号会被 Carvana 完全淹没。

Quiz

Q1: 13F 报告里的 value_x1000 字段值为 12345，这代表多少美元？

点击查看答案

12,345,000 美元（即 $12.345M）

value_x1000 的语义是"以千美元为单位"。所以值 12345 = 12345 × $1,000 = $12,345,000 ≈ $12.3M。常见错误是直接展示成 "$12,345" 或反过来除以 1000 显示成 "$12.345"，分别差 1000 倍和 1,000,000 倍。

Q2: 一个机构在 CVNA 的股数本季度从 100 万股变成 100 万股，市值从 $1 亿涨到 $1.5 亿。这算"加仓"吗？

点击查看答案

不算加仓，是被动市值上涨。股数没变 = 没有主动买卖。市值变动完全来自 CVNA 股价从 $100 涨到 $150。在 Top Buyers 排名里，应该按 股数变化（shares delta） 排序，而不是市值变化，否则会把"持有不动 + 股价上涨"误判为"主动加仓"。

Q3: 13F 能不能告诉我们"对冲基金在 CVNA 上的做空仓位"？

点击查看答案

不能。13F 只披露多头（long positions），不披露做空、不披露期权。要看做空仓位需要另外的数据源（如 S3 Partners、Markit Short Interest、Finra 双周短仓报告）。这也是为什么 IHS Markit 报告会单独标注 "data limitations" 提醒读者注意 13F 的盲区。
