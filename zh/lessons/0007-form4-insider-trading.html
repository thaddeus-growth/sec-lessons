Form 3/4/5 与内部人交易
高管在用脚投票，怎么看他们的信号

Lesson 0006 讲了外部机构（Vanguard、BlackRock 等）在买什么、卖什么。这一课讲公司内部人（CEO、CFO、董事）自己在买什么、卖什么。

内部人交易数据是 SEC 披露体系里信号噪音比最高的一种——因为内部人对公司的了解远胜外部投资者，他们的买入或卖出通常比机构持仓变动更有信息量。但前提是读懂，否则会被误判误导。

内部人交易不是"事件"——它是一条持续的数据流。单笔交易不说明问题，但加权后的整体情绪（InsiderSentiment 指标）能告诉你"这家公司的高管们对未来是怎么想的"。

Section 16 是什么、覆盖谁

Section 16 是 1934 年证券交易法的第 16 条，专门规范公司内部人的持股和交易披露。三个关键边界：

管辖对象含义具体例子
Officer（高管）公司高管，通常是 VP 及以上CEO、CFO、COO、各 SVP/EVP
Director（董事）董事会成员独立董事、控股母公司派出的董事
10% Owner（10% 以上股东）持有公司 >10% 股份的股东创始人、控股母公司、大型被动股东

⚠️ 中国 ADR 公司的盲区

Section 16 只适用于持有美国注册证券的内部人。中国 ADR 公司（优信、汽车之家）的高管通常持有开曼/VIE 层面的权益——这些权益不受 SEC Section 16 管辖。

结果：UXIN/ATHM 几乎没有 Form 4 数据。这不是"没有内部人交易"，而是"内部人交易不在 SEC 披露范围"。我们的系统在 config.py 里标记 has_section16 = False，Dashboard 显示"不适用"而非"无数据"。

三张内部人表格的区别

内部人交易披露有三张表，提交节奏和内容不同。简单区分：

表格全称什么时候提交内容
Form 3Initial Statement成为内部人后 10 天内初始持股声明（一次性）
Form 4Statement of Changes交易后 2 个工作日内每次买卖都要报
Form 5Annual Statement财年结束后 45 天内年度汇总（豁免交易）

Form 4 是最重要的——时效性最强（2 个工作日）、覆盖所有交易。Form 3 和 Form 5 主要是"补登"性质。

Form 4 的结构：Table I vs Table II

Form 4 由两张表组成，区别很关键： 

Table I — Non-Derivative Securities（直接持股的买卖）
  字段:
    - Title of Security          股票名称
    - Transaction Date           交易日
    - Transaction Code           P=买入, S=卖出, G=赠与, A=授予, F=交税卖出
    - Amount                     股数
    - Price                      单价
    - Amount After               交易后持股数

Table II — Derivative Securities（期权/derivatives 的变动）
  字段类似，但单位是"期权份数"而非"股数"
  常见 Code: M=行权, A=授予（期权）, C=转换

常见的交易代码（Transaction Code）

Code含义有信号吗？
P主动买入 (Purchase)✅✅✅ 强正面信号——内部人自掏腰包
S主动卖出 (Sale)✅ 弱负面信号——可能是分散持仓，不一定是看空
F交税卖出 (Tax withholding)❌ 几乎没信号——期权行权时预扣税是标准操作
G赠与 (Gift)❌ 弱信号——慈善/家族信托常见
A授予/行权 (Award/Exercise)❌ 没信号——薪酬的一部分，不是主动交易
M期权行权看行权后的卖出行为（Code F/S 配对）

最常见的误判

把"Code F 大量交税卖出"误读为"内部人看空"。这是最大的坑——高管在 10b5-1 计划下批量行权期权时，会同时产生大量 Code F 交易（预扣税）和 Code S 交易（卖出）。

正确做法是：看 Code P（主动买入）的金额，以及Code S 扣除行权配套卖出后的净卖出。直接 SUM 所有卖出代码会把信号淹没在噪音里。

内部人情绪指标的正确算法

简单数买卖笔数没意义（10b5-1 计划会产生大量配对交易）。正确做法是加权打分： 

InsiderSentiment = Σ(买入金额_i × 角色权重_i × 交易代码权重_i)
                  - Σ(卖出金额_j × 角色权重_j × 交易代码权重_j)

角色权重:
  CEO / CFO                   2.0    最高决策层
  10% Owner                   1.5    大股东
  Director                    1.0    董事会
  EVP / SVP / VP              0.8    执行层

交易代码权重:
  P (主动买入)                1.0
  S (主动卖出)                1.0
  F (交税卖出)                0.2
  G (赠与)                    0.3
  A (期权授予)                0.0
  M (期权行权)                0.0   (只看向行权后的卖出)

归一化到 [-100, +100]:
  > +30  → 看多信号
  -30 ~ +30 → 中性
  < -30  → 看空信号

注意 F 权重是 0.2 而不是 0——保留一点点信号（大量交税卖出可能反映大规模行权 = 集中套现），但不主导结论。

避免一个常见错误：在 10-K 发布前 1 个月，会出现大量内部人卖出（10b5-1 计划预设的）。这不一定是"内部人对年报没信心"——可能是按计划在 10b5-1 框架下分散持仓。要看 10b5-1 计划的备案日（一般在 8-K 的 Item 8.01 或 DEF 14A 里），不是看交易本身。

Form 144 — 减持的"预告片"

Form 144 是另一类内部人信号——它是事前通知，不是事后披露。

Form 144 的关键事实
触发条件：内部人计划出售 ≥5,000 股 或 ≥$50,000
提交时机：计划卖出前的 1-3 个工作日
有效期：计划卖出需在 Form 144 提交后 90 天内完成
aggregate market value：基于提交日股价的预估值（不是实际成交价）

关键解读：

  • Form 144 不等于"已经卖了"——它只是"准备卖"。计划可能不执行
  • 但多份 Form 144 连续提交 = 持续减持信号，要警惕
  • 用 Form 144 预判卖压时，要看"实际执行率"（OpenInsider 网站有这个数据）

中国竞品的特殊处理（实操规则）

我们的 5 家竞品里，优信 (UXIN) 和汽车之家 (ATHM) 是中国 ADR。它们没有 Form 4 / Form 144 数据，但有母公司层面的港股/A股披露。

系统处理规则

1. config.py 标记：has_section16 = False
2. Dashboard 显示：
    - 不是"无数据"，而是"不适用"（避免被误解为采集失败）
    - 旁边注明："内部人持有 VIE 层面权益，不受 SEC 披露约束"
3. 异常告警：
    - 如果中国 ADR 突然出现 Form 4 → 红色预警
    - 这是罕见事件，说明内部人特意把权益从开曼迁到美国层面来交易
    - 历史上这种情况极少见，但一旦出现，几乎总是有大事发生

Quiz

Q1: 高管在 10b5-1 计划下集中行权期权，触发了 100 笔 Code F（交税卖出）交易。你的系统应该怎么解读？

点击查看答案

几乎不解读为负面信号——这是 10b5-1 计划的标配操作。Code F 的权重是 0.2，100 笔 × 0.2 = 20 单位的噪音贡献。如果只看笔数会得出"内部人大规模减持"的错误结论；如果按加权算法，结论可能是中性或微负。判断时必须看该 10b5-1 计划是在什么时候备案的（通常在 8-K Item 8.01 或 DEF 14A 里），备案时间如果在 3-6 个月前，那基本是按计划行事。

Q2: 优信 (UXIN) 的 Dashboard 显示"内部人交易：不适用"，这和数据采集失败怎么区分？

点击查看答案

两者要明确区分，因为意义完全不同。"不适用" = 公司结构决定不适用 Section 16 披露（中国 ADR 内部人持有 VIE 层面权益），系统在 config.py 标记 has_section16 = False。"数据为空" = 采集失败或该期间没有 Form 4（罕见，因为 Form 4 是 2 个工作日强制披露）。Dashboard 的 UI 设计上要用不同的颜色和文字（"不适用" 用灰色，注释说明原因；"数据为空" 用红色，可能提示 API 故障）。

Q3: 看到 Carvana 的 CFO 在 10b5-1 计划下卖了 $5M 股票，但同时 VP 主动买入 $1M。你的系统怎么打分？

点击查看答案

要看这 $5M 卖出的具体 Code 是什么。如果是 Code F（交税卖出）+ 配套的 Code S（10b5-1 计划下的卖出），那么权重 = 0.2 + 1.0 × 角色权重 2.0 = 2.2 单位的负向贡献。但 VP 主动买入 (Code P) = 1.0 × 0.8 = 0.8 单位的正向贡献。如果金额差 5:1，结果是净负向 ~-3,400 单位，但归一化后可能落在 -30 ~ 0 区间，结论是"谨慎"，不是"看空"。所以即使金额大，但因为是 10b5-1 计划下的程序化卖出，结论要打折。
